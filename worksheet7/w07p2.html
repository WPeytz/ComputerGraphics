Worksheet 7 - Environment Mapping and Normal Mapping - Part 2

Environment Background Implementation

This implementation renders a reflective sphere against an environment background using cube map textures. The same shader handles both objects using the Mtex transformation matrix.

Key Concept: The Mtex Matrix

The u_mtex uniform transforms vertex positions into texture coordinates for cube map lookup. This allows us to use the same shader for different purposes:

For the SPHERE:
Mtex = Identity Matrix
  • Vertex positions are already normalized (unit sphere)
  • Position vector = Normal vector (for sphere centered at origin)
  • The surface normal points in the direction to sample from the environment
  • Result: Creates realistic reflections of the environment
For the BACKGROUND QUAD:
Mtex = inverse(viewRotation) × inverse(projection)
  • Vertices are in clip space at corners: (-1,-1,0.999) to (1,1,0.999)
  • Transforms clip space positions → world space directions
  • Result: Displays the environment map as a background

Background Quad Transformation Pipeline

The background quad undergoes these transformations:

Step 1: Start with clip space position
Vertex: (x, y, 0.999, 1) where x,y ∈ [-1, 1]

The z = 0.999 places it near the far plane, behind all other geometry.

Step 2: Apply inverse(projection)
Clip space → Camera space

Undoes the perspective projection, giving us a direction ray from the camera through each screen pixel.

Step 3: Apply inverse(viewRotation)
Camera space → World space

Rotates the direction back to world coordinates. We only use rotation (no translation) because the environment is at infinity.

Step 4: Cube map lookup
texture(u_texture, worldDirection)

The resulting world space direction is used to sample the appropriate face of the cube map.

Rendering Order

  1. Background Quad first: Drawn at z=0.999 (far plane)
  2. Sphere second: Drawn with normal depth testing

This ensures the sphere appears in front of the environment, with proper depth occlusion.

Why This Works

By using Mtex, we avoid needing separate shaders or special texture coordinate attributes. The same vertex shader can handle both cases:

  • For the sphere: Mtex = I → uses position as normal
  • For the background: Mtex = transforms to world directions

The fragment shader simply looks up the cube map using the interpolated direction vector, creating seamless environment mapping for both objects.
