Projection Shadows (Part 2)

Advantages:

  • Simple to implement - Only requires basic matrix math (shadow projection matrix)
  • Fast computation - Single rendering pass, no extra textures needed
  • Low memory usage - No shadow map texture storage required
  • Minimal shader complexity - Simple vertex transformation with projection matrix
  • Works well for planar receivers - Perfect for flat ground planes

Disadvantages:

  • No self-shadowing - Objects cannot cast shadows on themselves
  • Limited to planar surfaces - Only works for flat shadow receivers (like ground)
  • No soft shadows - Sharp, hard-edged shadows only
  • Depth fighting artifacts - Requires epsilon offsets and special depth testing (greater)
  • Not perspective correct - Shadows can look distorted at angles
  • Cannot handle complex geometry - Fails with curved or non-planar receivers
  • Single receiver limitation - Shadows only appear on one designated plane

Shadow Mapping (Parts 3-4)

Advantages:

  • Handles self-shadowing - Objects can correctly shadow themselves
  • Works on any geometry - Shadows appear on curved surfaces, complex meshes
  • Multiple shadow receivers - Any object in the scene can receive shadows
  • More realistic appearance - Proper depth-based occlusion
  • Supports multiple lights - Can create multiple shadow maps for different lights
  • Hardware support - Built-in GPU features (comparison samplers, PCF filtering)
  • Extensible - Can add soft shadows, cascaded shadow maps, etc.

Disadvantages:

  • Higher complexity - Requires two rendering passes and coordinate transformations
  • Memory overhead - Needs large depth texture (e.g., 2048x2048 or higher)
  • Resolution dependent - Shadow quality limited by shadow map resolution
  • Shadow acne artifacts - Requires bias values to prevent self-shadowing errors
  • Peter Panning - Too much bias can cause shadows to detach from objects
  • Performance cost - Two render passes instead of one
  • Aliasing issues - Can show jagged edges, especially at grazing angles
  • Limited shadow distance - Shadow map covers finite area (frustum)

Summary

Projection shadows are ideal for simple scenes with flat ground planes where performance and simplicity are priorities, and self-shadowing is not needed. They excel in scenarios like a character walking on flat terrain, where the computational overhead needs to be minimal.

Shadow mapping is the industry-standard technique for realistic shadows in 3D graphics, handling complex geometry and self-shadowing at the cost of increased complexity and memory usage. It is used in virtually all modern games and 3D applications where visual fidelity is important.

For most modern 3D applications, shadow mapping is preferred despite its disadvantages because it provides much more realistic and flexible shadow rendering. The additional computational cost is acceptable given the significant improvement in visual quality and the ability to handle arbitrary scene geometry.
